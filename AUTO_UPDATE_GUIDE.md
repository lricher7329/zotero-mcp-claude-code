# Zotero MCP Plugin Auto-Update Guide

## Auto-Update Configuration

The plugin is configured for Zotero add-on updates through GitHub releases in
`lricher7329/zotero-mcp-claude-code`.

The built add-on manifest declares:

```json
{
  "applications": {
    "zotero": {
      "id": "zotero-mcp-plugin@lricher7329.github.io",
      "update_url": "https://github.com/lricher7329/zotero-mcp-claude-code/releases/latest/download/update.json",
      "strict_min_version": "6.999",
      "strict_max_version": "9.*"
    }
  }
}
```

## Release Assets

The release workflow uploads the scaffold build outputs:

- `.scaffold/build/zotero-mcp-for-claude-code.xpi`
- `.scaffold/build/update.json`

Those two filenames must stay aligned. The update manifest should point at:

```text
https://github.com/lricher7329/zotero-mcp-claude-code/releases/download/vX.Y.Z/zotero-mcp-for-claude-code.xpi
```

The checked-in `zotero-mcp-plugin/update.json` is a local manifest generated by
`npm run prepare-release`. The actual release upload uses the build-generated
`.scaffold/build/update.json`, which includes the XPI hash produced by that
specific build.

## Release Process

From `zotero-mcp-plugin/`:

```bash
npm version patch --no-git-tag-version
npm run prepare-release
npm run build
npm run test:unit
npm run lint:check
```

Also update:

- `src/modules/httpServer.ts` `SERVER_INFO_VERSION`
- root `README.md` version badge and tool counts, when applicable
- release notes or issue references, when applicable

Commit from the repository root, tag the release, and push:

```bash
git add .
git commit -m "chore: release vX.Y.Z"
git tag vX.Y.Z
git push origin main --tags
```

The root workflow at `.github/workflows/release.yml` builds the XPI and uploads
the matching build-generated update manifest.

## Testing Auto-Update

1. Install an older plugin version.
2. In Zotero, open `Tools > Add-ons`.
3. Use the gear menu and choose `Check for Updates`.
4. Confirm Zotero fetches `update.json` from the latest GitHub release and then
   downloads `zotero-mcp-for-claude-code.xpi`.

Manual checks:

```bash
curl -L "https://github.com/lricher7329/zotero-mcp-claude-code/releases/latest/download/update.json"
curl -I "https://github.com/lricher7329/zotero-mcp-claude-code/releases/download/vX.Y.Z/zotero-mcp-for-claude-code.xpi"
```

## Troubleshooting

- Update not detected: confirm the installed version is lower than the release
  version and that `latest/download/update.json` is reachable.
- Download fails: confirm the XPI filename in `update_link` matches the uploaded
  release asset exactly.
- Version mismatch: confirm `package.json`, `SERVER_INFO_VERSION`, and
  `update.json` all use the same stable version.

## Zotero 9 HTTP Compatibility

Version 1.8.5 changes the local HTTP request reader to wait for Gecko stream
readiness callbacks instead of relying on repeated `available()` polling. This
addresses GitHub issue #1, where Zotero 9 could accept TCP connections on
`127.0.0.1:23120` but return zero bytes until MCP clients timed out.
